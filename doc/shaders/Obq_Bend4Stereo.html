Obq_Bend4Stereo

|  Description  |  Installation  |  Render Tree Usage  | Copyrights  |

Category: Texture

Shader Family: Texture

Output: Color

Obq_Bend4Stereo is a passthrough node that changes the ray direction for camera rays. Used with stereo cameras, this allows the shading of the two cameras to be calculated from a single one.

Bend Mode

Bend Mode How to bend :
  • Don't bend : leave ray direction as-is.
  • Bend from left : both cameras will be calculated as if coming from the left camera.
  • Bend from right : both cameras will be calculated as if coming from the right camera.
  • Bend from center : both cameras will be calculated as if coming from the center camera.


Cameras

Center The center camera. 
Left The left camera.
Right The right camera.
CameraStatus Tells if all cameras are valid.

The pick order is center, left, right.


Installation

  1. Copy the Obq_Bend4Stereo.dll in the proper "SItoA\Application\bin\nt-x86-64" folder and the ObqBend4Stereo.spdl in the proper "SItoA\Application\spdl" folder.
  2. You'll need to install Microsoft Visual C++ 2010 redistributable package (x64) if not already done.


Render Tree Usage

Use the Obq_Bend4Stereo on all materials as a passthrough right before the "Surface" port of the material node or the "camera" port of a "ray_switch" node. In this stereo frame, the magenta and green in the highlights represent disparities between the render with and without bending.

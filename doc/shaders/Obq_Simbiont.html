Obq_Simbiont

|  Description  |  Installation  |  Render Tree Usage  |  Copyrights  |

Category: Texture

Shader Family: Texture

Output: Color

Obq_Simbiont is a Darktree 2.5 loader node in Softimage that reads .dsts files for the Arnold Renderer.

Darktree

Darktree file The darktree file you want to load.
Note that you can change the default browsing directory on line #645 of the Obq_Simbiont.spdl.
Darktree component libs path This is the path to the DarkTree Engine "dte_components" libraries. If this path is incorrect, you will see red Xs as the resulting texture.
Note that you MUST change this path accordingly on line #30 of the Obq_Simbiont.spdl.


Output

RGB What is outputed in the RGB channels. This is based on the loaded file and all choices might not return results.
Choices are :
  • Black
  • Color [default]
  • Percent
  • Elevation [displacement or bump]
  • Diffuse level
  • Diffuse Func
  • Luminosity
  • Specular Level
  • Specular Func
  • Specular Color
  • Glossiness
  • MetalLevel
  • Anisotropy
  • Aniso Dir
  • Reflectivity
  • Environement
  • Transparency
  • Refraction
  • Clear Coat Level
  • Clear Coat Glossiness
  • Clear Coar Thickness
  • Clear Coat Smoothness
  • Bump [Normal map]
  • Alpha
Grayscale If the choice selected for RGB returns a single value, elevation for instance, it will be put in the red channel for simplicity. If you want to copy that value in all the channels, you can check this option.
Alpha Like RGB, but the result is stored in the alpha channel.


Bump

Sample Size A rough indication of the diameter of the point/pixel being sampled.  This is used when evaluating bump.
Bump Scale A bump height scaling factor.  This is used when evaluating bump.


Camera

Camera Near Some darktree files have effects based on a normalized value of how far is the camera.
Use the near and far to give the range for the effect.
Camera Far Same as above.


Others

Continuous Rep A 0.0 to 1.0 normalized value used for function evaluation. DTE_EVAL_DIFFUSE_FUNC and DTE_EVAL_SPECULAR_FUNC are where this value is often used when evaluating a texture. If you need additional information on using the specular and diffuse functions, please contact DarkSim support.
Region Rep A random integer value used to indicate different regional areas of the texture. Patterned components within DarkTree, Bricks for example, set this value for linked child components. Control components use this value to generate random colors and values. A potential use for this value would be a custom designed texture that requires regional information to be passed into the texture from the renderer.


Animation

Time Can be seen as a current frame number for an animated texture. This is a floating point value and need not be an integral frame.  Please note that this will have an effect only on DarkTrees that use the Time component.


Coordinates

Coordinate System This is the system used to evaluate the textures.
Choices are :
  • Object Space [default]
  • World Space
  • UV Space [first uv set]
  • Camera Space [camera projection]
  • Raster Space [XY Screen coordinates]
  • Custom [enables Custom Coordinates]
Custom Coordinates This is most often used to connect a node such as a "Projection Lookup".


Transform

Scale This is a global scaling factor.


Acceleration

Tweak at global scope Tweaks are perform with the given value for all the object. This is good when you don't need to drive tweaks with expressions or textures. Otherwise, it should be off.


Tweaks #X

Enable Enable this tweak.
A given DarkTree file may contain any number of macro controls, or Tweaks. These tweaks are evaluated at runtime and can be modified with expressions or texture maps in Softimage. There is a maximum of 16 tweaks in this implementation.
Name The name of the tweak.
The tweaks are automatically loaded by a python script when the filename is change. Only the proper input choice will be displayed.
Color A color input for the tweak.
Used only for color driven tweaks.
Float A scalar input for the tweak.
If the tweak is of boolean nature, x >0 => true.
If the tweak requires an integer, the scalar will be rounded.


Installation

  1. Copy the Obq_Simbiont.dll in the proper "SItoA\Application\bin\nt-x86-64" folder and the Obq_Simbiont.spdl in the proper "SItoA\Application\spdl" folder.
  2. You'll need to download the DarkTree Engine (DTE Win 64-bit) , copy the "components" folder in the proper folder "..\SItoA\Application\bin\nt-x86-64" and change the path in the Obq_Simbiont.spdl on line 30 to the proper "components" path.
  3. You also have to copy the "dte_2600_64.dll" in "C:\Windows", unless you have the full DarkTree software installed.
  4. Optionally, you can change the path to the "DTRep_thumb" ( DarkTree repository ) on line 645 of the Obq_Simbiont.spdl.
  5. You'll need to install Microsoft Visual C++ 2010 redistributable package (x64) if not already done.


Render Tree Usage

The simplest way to use the Obq_Simbiont, is to open a darktree file, have the "Color" output in RGB to drive the Kd_color of the Standard shader and "Elevation" in Alpha in order to drive a bump3d node. Unfortunately, because there is no node caching nor multioutput nodes in Arnold yet, this will result in 2 evaluations of the Obq_Simbiont, but nothing can be done easily as of now.

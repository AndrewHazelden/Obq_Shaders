SItoA :

Copy the content of the "softimage\spdl" folder into "Addons\SItoA\Application\spdl".

Copy the content of "softimage\compounds" into "Addons\SItoA\Data\RTcompounds".

Copy the .dll in "bin\vXXX" into "Addons\SItoA\Application\Plugins\bin\nt-x86-64". [Windows]
Copy the .so in "bin\vXXX" into "Addons/SItoA/Application/Plugins/bin/linux64". [Linux]
Copy the .dylib in "bin\vXXX" into "Addons/SItoA/Application/Plugins/bin/??". [OSX]

MtoA (not yet fully integrated):

Copy the content of "maya\metadata" into "solidangle\mtoadeploy\<Maya Version>\shaders".

Copy the content of "maya\ae" into "solidangle\mtoadeploy\<Maya Version>\scripts\mtoa\ui\ae".

Copy the content of "maya\icons" into "solidangle\mtoadeploy\<Maya Version>\icons".

Copy the .dll in "bin\vXXX" into "solidangle\mtoa\<Maya Version>\shaders". [Windows]
Copy the .so in "bin\vXXX" into "solidangle/mtoa/<Maya Version>/shaders". [Linux]
Copy the .dylib in "bin\vXXX" into "solidangle/mtoa/<Maya Version>/shaders". [OSX]

"Core" version :

The "Core" version contains all Obq_Shaders that don't require additional libs than Arnold's.
Here's the list of the shaders you won't find in the "Core" version and must be compiled/copied separately:

    * Obq_Simbiont


To use Obq_Simbiont :    [Windows]

Copy the "bin\dte\dte_2600_64.dll" (DarkTree library) to either "C:\Windows" or another arbitrary directory (AD).

    If you chose the AD method, you'll need to append this AD to the PATH environment variable.
    For example, you can add :
    "set PATH=%PATH%;\\serverX\dte_folder" in the .bat that launches Softimage (without the quotes).

Copy the "\bin\dte\dte_components" folder somewhere and change the path on line 30 of the Obq_Simbiont.spdl (the "componentLibsPath" parameters).

If you don't own a copy of DarkTree, you can download some DarkTree files here :  http://www.darksim.com/html/downloads.html

     ** Because Obq_Simbiont requires a Windows library, it is not available for Linux and Mac OSX. **
